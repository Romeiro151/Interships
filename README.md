# Interships
This project consists of a relational database system designed to manage student internships. The database was implemented in SQL, following proper database normalization principles to ensure data integrity and reduce redundancy.

The system includes multiple related tables (e.g., students, companies, internships, supervisors) connected through primary and foreign keys, enforcing referential integrity constraints. SQL queries were implemented for data retrieval, filtering, aggregation, and reporting. The project also makes use of JOIN operations, indexing, and structured schema design to optimize performance.

On the application side, a web-based interface was developed using HTML and PHP. PHP was used as the server-side scripting language to handle form submissions.

This project strengthened my understanding of:
  Relational database design
  SQL queries and data manipulation (DML)
  Schema definition and constraints (DDL)
  Backend integration with PHP
  Client-server architecture fundamentals
